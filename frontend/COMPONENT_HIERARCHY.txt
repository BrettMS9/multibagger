COMPONENT DETAILS
=================

ScoreGauge Component Props:
- score: number
- maxScore: number
- size: 'sm' | 'md' | 'lg'

Renders:
- SVG circle (background)
- SVG circle (colored progress)
- Center text (score / maxScore)
- Colors: green (75%+), yellow (50%+), orange (25%+), red (<25%)

FactorRow Component Props:
- label: string
- score: number
- maxScore: number
- rationale: string
- isImportant?: boolean (shows "MOST IMPORTANT" badge)
- isContrarian?: boolean (shows "CONTRARIAN" badge)

Renders:
- Label + Badges
- Score display
- Progress bar
- Rationale text

DATA FLOW
=========

1. Single Stock Analysis:
   User types ticker → TickerSearch
   → handleSearch() triggered
   → setSearchTicker() + setShouldFetch(true)
   → useStockScreen() hook activates
   → TanStack Query calls api.screenStock()
   → Fetch GET /screen/{ticker}
   → Response cached by TanStack Query
   → ScoreCard renders with data

2. Bulk Screening:
   User selects exchange → BulkScreener
   → setSelectedExchange()
   User clicks "Screen All" → setIsScreening(true)
   → useBulkScreen() hook activates
   → TanStack Query calls api.bulkScreen()
   → Fetch GET /bulk-screen/{exchange}
   → Response cached by TanStack Query
   → Results table renders

STATE MANAGEMENT
================

App-level State:
- activeTab: 'single' | 'bulk'
- searchTicker: string
- shouldFetch: boolean

BulkScreener State:
- selectedExchange: 'NSE' | 'BSE' | 'NASDAQ' | 'NYSE'
- isScreening: boolean

TanStack Query manages:
- Loading states
- Error states
- Data caching
- Refetch logic
- Stale time (5 min single, 10 min bulk)
